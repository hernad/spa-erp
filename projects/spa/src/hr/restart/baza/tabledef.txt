# Automatsko punjenje tablice u ovaj format je jednostavno:
# u SQL pilotu se ode na popis tablica i odabere tablica za koju se traži
# definicija u ovom formatu. I voila! Definicija se nalazi u clipboardu i
# može se jednostavno pejstat u ovaj fajl. Neat, eh?

# Definicije za sve tablice u bazi. Linije koje pocinju sa # su komentari.
# Pojedina definicija pocinje s $TABLENAME a zavrsava krajem datoteke
# ili definicijom sljedece tablice. Svaka kolona tablice odredjuje se
# jednim redom ove datoteke, po formatu:
# NAME;TYPE;PRECISION;INDEX;DEFAULT;CAPTION<;WIDTH>
# NAME je ime kolone u modulu i u bazi.
# TYPE moze biti 'alpha', 'text', 'char' (tekst polja, String),
# 'timestamp', 'date' (datum, Timestamp), 'numeric', 'decimal' (realni brojevi, BigDecimal),
# 'integer', 'int', 'short' (cijeli brojevi, int ili short)
# PRECISION je broj znakova za tekst polja, sirina i broj decimala za numericka polja.
# INDEX moze biti 'pkey', 'key' (dio primarnog kljuca), 'yes', 'idx', 'uidx' (indeksirano) 
# ili nista.
# DEFAULT moze biti prazan string ili inicijalna vrijednost polja.
# CAPTION je duzi opis polja
# WIDTH je opcionalna sirina polja, ili '*' ako polje treba biti nevidljivo.

# iza svih kolona moze se rucno dodati jos koji index, po formatu
# IDX=ime kolone, ili UIDX=ime kolone. Prazni redovi u ovoj datoteci se ignoriraju.
# Za svaku tablicu mora postojati i klasa u hr.restart.baza, u kojoj treba biti samo
# osnovna funkcionalnost (dohvacanja static instance te pripadajuci shared datasetovi),
# te negdje u konstruktoru poziv super.initModule(). 
# Za svaku takvu klasu treba dodati i njeno ime u ConsoleCreator.moduleClasses array.
# Tako se nista ne mora mijenjati u vanjskim klasama, i dalje rade stvari poput 
# Agenti.getDataModule().getTempSet() itd.
# Ukoliko za neki modul nema definicije u ovoj datoteci, onda mora postojati stari
# oblik inicijalizacije (onaj s hrpom poziva tipa KOLONA.setBlaBla(nesto)).
